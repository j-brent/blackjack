## Phase 4: Bug Fixes — Issues 1, 2, 5 (15:31 – 16:14, ~43m)

### ISSUE-001: Suit symbols garbled in CLI (15:31, ~7m)

**Commit** `6fb1716` — *Fix ISSUE-001: Set UTF-8 console output on Windows for suit symbols*

The CLI displayed garbled Unicode suit symbols on Windows. Root cause: Windows console defaults to a legacy code page. Fix: call `SetConsoleOutputCP(CP_UTF8)` at startup.

### ISSUE-005: C-style (void) casts (15:31 – 16:08, ~37m)

**Commit** `e0370f9` — *Fix ISSUE-005: Replace C-style (void) casts with proper result handling*

Code review finding. The codebase used `(void)expr` to discard `[[nodiscard]]` return values. Replaced with `std::ignore = expr` per C++ Core Guidelines. Fixed on a separate branch by the user and merged back.

### ISSUE-002: Split appears to skip player turn (16:08 – 16:13, ~5m commit, ~1h investigation)

**Commit** `70389c2` — *Resolve ISSUE-002: Split skips player turn (could not reproduce)*

The user reported that splitting a non-ace pair in the CLI jumped straight to results. Deep investigation:
- Static analysis of game logic — no code path found that skips player turn for non-aces.
- Brute-force test: 100,000 random seeds (5,145 non-ace splits), zero failures.
- Added temporary seed CLI argument to reproduce specific hands.
- Found and fixed a secondary bug: the CLI's if-else chain checked `'s'` (stand) before `"split"`, so typing "split" would trigger stand.

Resolved as "could not reproduce." Hypothesis: the original observation was due to garbled Unicode (pre ISSUE-001 fix) making it look like the turn was skipped, or the hand was actually aces (which auto-stand by rule).

## Phase 5: TUI Overhaul — Issues 3 & 4 (16:14 – 16:44, ~30m)

**Commit** `a3c1139` — *Fix ISSUE-003 and ISSUE-004: TUI dynamic controls and quit button*

Two related TUI bugs:
- **ISSUE-003**: No way to start a new game after a round ends.
- **ISSUE-004**: Quit action undiscoverable (only Escape worked, no visible button).

Root cause: `build_controls()` was called once at startup and never rebuilt. Buttons were static.

Fix — major rewrite of the TUI frontend:
- Removed the one-shot `build_controls()` function.
- Created all buttons inline with FTXUI `Maybe` wrappers for dynamic show/hide based on game state.
- `(Q)uit` button always visible (no Maybe wrapper).
- `(N)ew Game` button appears when round is over or at initial deal.
- Added `play_dealer_if_needed()` helper — also fixed a latent bug where the dealer turn was skipped after hit-bust and ace-split.
- Changed `std::ignore` to `assert()` for invariant documentation.
- Migrated to `std::ranges::find` per new C++ coding directive.

## Phase 6: Codebase-Wide std::ranges Migration — Issue 6 (16:44 – 16:49, ~5m)

**Commit** `0d9a4f2` — *Fix ISSUE-006: Replace iterator-pair std algorithms with std::ranges*

Created during ISSUE-003/004 work as a follow-up task. Replaced all 4 remaining iterator-pair algorithm calls in `cli/src/main.cpp`:
- `std::find` → `std::ranges::find`
- `std::find_if_not` → `std::ranges::find_if_not` (with `std::views::reverse`)
- `std::transform` → `std::ranges::transform`

Also added the std::ranges preference as a permanent learning in the C++ knowledge base.
