## Phase 5: Web Frontend — Design Iteration (this session, 12:00 – 14:52)

### 5a. Five Design Feedback Items (carried from previous session context)

Work continued from a previous conversation that ran out of context. Five design changes were planned and implemented:

1. **SVG cards** — Rewrote `card-renderer.js` from styled `<div>` elements to inline `<svg>` with `viewBox 0 0 250 350`. Resolution-independent, scalable via CSS em units.
2. **Card overlap** — Removed `overflow-x: auto`, added negative-margin fan overlap so cards never scroll.
3. **Static buttons** — Pre-rendered Hit/Stand/Split/New Game in HTML with `hidden` attribute toggled per state. No more DOM recreation on each render.
4. **AT-8 test** — Card display correctness test (valid rank/suit text, no Embind artifacts).
5. **UX spec update** — Converted all px values to rem/em throughout `blackjack-web-app.md`.

### 5b. Overlap → Diagonal Cascade

User provided reference image of real blackjack dealing. Cards should cascade diagonally (right and down), not just overlap horizontally.

- Added `--card-overlap-visible: 1.5em` and `--card-cascade-down: 1em` CSS tokens
- Rewrote `applyCardOverlap()` for always-on diagonal cascade
- Reversed dealer card render order (hole card underneath in DOM, up card on top)

### 5c. Bug Fixes

- **White face-down card** — CSS `.card-bg { fill: white }` overrode the SVG `fill="url(#card-back-stripes)"` pattern attribute. Fixed with `.card.face-down .card-bg` override rule.
- **Bottom-right text clipping** — Rotated text at `rotate(180, 235, 315)` extended past the viewBox boundary. Fixed by rotating around the card center `rotate(180, 125, 175)` using same coordinates as top-left text.
- **Player card right-side clipping** — Container `justify-content: center` pushed overflowing flex items outside bounds. Changed to `align-items: flex-start`.
- **Split hands overflow** — Changed `#player-area-split` from `flex-direction: column` to `row` at all viewports.
- **SVG font-size px violation** — Moved font-size from CSS (px) to SVG attributes (unitless viewBox coordinates) to comply with no-px standard.
- **Font sizing** — Iterated corner rank/suit from 28 → 42 → 56 → 84 → back to 56. Center suit 80 → 120. User settled on corners=56, center=120.

### 5d. New Acceptance Tests

- **AT-9** — SVG text within viewBox bounds. Asserts all `<text>` bounding boxes stay inside 0-250 x 0-350.
- **AT-10** — Visual containment at 360x640. Walks parent-child pairs (cards in containers, containers in areas, areas in main, main in viewport) asserting no overflow. Runs during initial deal and after split.

### 5e. Layout Shift Fix

**Commit** `076befa` — *Fix button area layout shift during dealer turn*

When all buttons are hidden between player finish and dealer finish, the `#actions` container collapsed, causing a visible vertical jump. Fixed with `min-height: 4.75rem` on `#actions`.
